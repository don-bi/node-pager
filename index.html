Node is an asynchronous event-driven JavaScript runtime designed to build scalable network applications.

History

  • JavaScript is compiled in browsers by a JS engine. There are many such as V8 and Spidermonkey, but V8 is the most popular engine and is used by chrome.
  • Since V8 was a super fast engine, people used it to create a runtime environment for JS, which is Node.
  • Node isn't a programming language, nor a framework, it’s a runtime environment for JS.

Asynchronous Nature

  • Allows tasks to be run simultaneously without waiting for one to end before commencing another.
  • If Node receives another event while one is running, it sends it to another worker thread to perform the new process.

Event-Driven Nature

  • When an event is triggered in Node, a callback function (just calling another function) occurs in response to that event.
  • This is so useful because they require significantly less resources than normal functions since they aren’t called when Node first sees it, but once an event happens some point in the future.
  • Takes much less of a toll on the CPU.

WHat ARE MODULES?

  • Modules are similar to Python and Java libraries, and they basically contain different built-in functions of those modules that could be used after calling them.
  • The syntax to call them is by setting a variable to require the module.
  • const express = require('express');

  • The GOAT of all time.
  • Short for the Node Package Manager, which is one of the most important parts of Node.
  • The NPM has a ton of packages and modules built by the community to use.
  • Knowing different types of packages and how to use them efficiently will streamline your code a lot more and also allow you to do some really cool stuff.
